# AGENTS.md

## Project

Next.js 16 + React 19 + Tailwind v4 + shadcn/ui (new-york style). AML monitoring dashboard generated by v0.app. Chinese-language UI.

## Commands

```bash
pnpm dev    # next dev — start dev server
pnpm lint   # eslint .   — WARNING: no eslint config exists, will likely fail
```

There is no `test` or `typecheck` script. TypeScript checks via `npx tsc --noEmit`.

## pnpm, not yarn

Despite both lockfiles existing, use **pnpm**. `pnpm-lock.yaml` is the authoritative lockfile.

## Build is intentionally broken

`"build": "vue-cli-service build"` in package.json is for Vue, but this is a Next.js project. For a Next.js build, use `next build` instead. The `postbuild` hook runs `compress.js` (empty file). `vite.config.js` with `vite-plugin-zip-pack` is vestigial and unused.

## Next.js config

`next.config.mjs` is auto-generated by v0. **Do not edit it.** User overrides go in `next.user-config.mjs`. The generated config enables Turbopack, allows `*.vusercontent.net` origins, and skips image optimization in dev.

## Architecture

```
app/
  page.tsx          → /          main dashboard
  assistant/
    page.tsx        → /assistant AI chat page (mock responses)
  layout.tsx        → root layout, zh-CN
  globals.css       → Tailwind v4 + shadcn theme tokens

components/
  aml-header.tsx     → top bar
  aml-sidebar.tsx    → collapsible left nav (6 sections, 8 sub-items under 可疑行为监测)
  aml-content.tsx    → main area with hardcoded mock case data
  aml-assistant-button.tsx → floating button, links to http://localhost:8080/ (external)
  theme-provider.tsx → UNUSED — wraps next-themes, not imported in layout
  ui/                → ~70 shadcn/ui primitives (Radix-based)

hooks/       → use-toast, use-mobile
lib/utils.ts → cn() helper (clsx + tailwind-merge)
```

## Gotchas

- **Duplicate files** (identical content): `hooks/` and `components/ui/` both contain `use-mobile.ts`/`.tsx` and `use-toast.ts`. When updating hooks, check both locations.
- **Duplicate CSS**: `styles/globals.css` is identical to `app/globals.css`. Only `app/globals.css` is used by the app. `styles/globals.css` is dead code.
- **Assistant button** links to `http://localhost:8080/` (hardcoded, not the internal `/assistant` route).
- **All data is hardcoded mock data**. No backend API, no database.
- **ThemeProvider is defined but not wired into the layout** — dark mode toggle won't work.
- **`.snowflake/` and `.v0-trash/`** are v0 sandbox internals, gitignored.
- **Tailwind v4** uses the `@tailwindcss/postcss` PostCSS plugin (not the v3 `tailwindcss` plugin).
- **When adding shadcn/ui components**, they install to `components/ui/` and registries are configured in `components.json`. All imports use `@/` path alias (e.g. `@/components/ui/button`).
